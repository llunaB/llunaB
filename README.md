## This is my convention.
```
Feat:     새로운 기능을 추가할 경우
Fix:      버그를 고친 경우
Style:    코드 포맷 변경, 간단한 수정, 코드 변경이 없는 경우
Refactor: 프로덕션 코드 리팩토링
Docs:     문서를 수정한 경우(ex> Swagger)
Rename:   파일 혹은 폴더명 수정 및 이동
Remove:   파일 삭제
Chore:    빌드 업무 수정(ex> dependency 추가)
Test:     테스트
Upload:   폴더 및 파일 
Update:   폴더 및 파일 업데이트
```
